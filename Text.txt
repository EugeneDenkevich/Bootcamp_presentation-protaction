Слайд 1:
Над проектом работали следующие бэкенд разработчики.

Слайд 2:
Проект написан с помощью фрэймворка джанго DRf с использованием следующих инструментов и библиотек.

Слайд 3:
Бэкенд-направлением была проделана следующая работа.
- Swagger сделали более читабельным и юзабельным.
- Реализовали гибкую работу с синхронизацией между БД
- Переписали старые тесты с ОРМ Джанги на SQLAchemy.
- Реализовали систему оповещения учителя об изменениях в расписании.
- Создали возможность изменить пароль пользователю.
- Произвели полный рефакторинг кода для последующей унификации проектов iTeen1 и iTeen2.
- Реализовали выдачу данных занятий преподавателю

Слайд 4:
Отдельно выделим реализацию расписания.
Для этого был создан эндпоинт, отдащий список занятий конкретного учителя за указанный период. 

На данный момент это месяц и неделя.
Но, архитектура кода позволяет в дальнейшем добавить любой другой период в качестве параметра.

Слайд 5:
Для этого нам не хватало данных и мы расширили класс синхронизации.
Теперь приложение подтягивает данные из базы данных CRM IT-Academy в том объеме, который нам нужен.
Дополнительно синхронизируются данные необходимые для последующих релизов.
Также данная функция формирует типы занятий и сообщения об изменениях в расписании.

В процессе были решены конфликты, связанные с датой и временем.
Так же были использованы сторонние библиотеки, позволяющие работать с датой и временем более гибко.
